# DevOps Lab

# Demonstration of Git Commands using GitBash and GitHub

## Aim

To demonstrate and create a project in local and remote repositories using **GitBash and GitHub** and apply the commands:
`git init`, `git status`, `git log`, `git add`, `git commit`, `git push`, `git config`, `git clone`, and `git reset`.

---

# Requirements

* Git installed on the system
* GitHub account
* GitBash terminal
* Internet connection

---

# 1. Configure Git

The `git config` command is used to configure the Git username and email.

### Syntax

```
git config --global user.name "YourName"
git config --global user.email "youremail@example.com"
```

### Example

```
git config --global user.name "Jayanth"
git config --global user.email "jayanth@gmail.com"
```

---

# 2. Create Local Repository using Git Init

The `git init` command is used to create a new Git repository in the local system.

### Syntax

```
git init Demo
```

### Example

```
mkdir Demo
cd Demo
git init
```

After running this command, a hidden **.git folder** will be created.

---

# 3. Git Status Command

The `git status` command shows the current state of the working directory and staging area.

### Syntax

```
git status
```

### Example Output

```
On branch master
No commits yet
Untracked files:
demo.txt
```

This shows files that are modified or not yet committed.

---

# 4. Create a File in Repository

Create a sample file.

```
echo "This is DevOps Lab File" > demo.txt
```

Check the status again.

```
git status
```

---

# 5. Git Add Command

The `git add` command adds files to the **staging area**.

### Syntax

Add a single file:

```
git add filename
```

Example:

```
git add demo.txt
```

Add multiple files:

```
git add *
```

---

# 6. Git Commit Command

The `git commit` command records changes permanently in the repository.

### Syntax

```
git commit -m "Commit Message"
```

### Example

```
git commit -m "Initial commit for DevOps project"
```

---

# 7. Git Log Command

The `git log` command displays the commit history.

### Syntax

```
git log
```

### Example Output

```
commit 7a8c2b...
Author: Jayanth
Date: Mon Mar 15

Initial commit for DevOps project
```

To display limited commits:

```
git log -3
```

---

# 8. Create Remote Repository on GitHub

1. Login to **GitHub**
2. Click **New Repository**
3. Enter repository name:

```
devops-lab
```

4. Click **Create Repository**

---

# 9. Connect Local Repository with GitHub

Add remote repository URL.

### Syntax

```
git remote add origin repositoryURL
```

### Example

```
git remote add origin https://github.com/username/devops-lab.git
```

---

# 10. Git Push Command

The `git push` command uploads local repository content to a remote repository.

### Syntax

```
git push origin master
```

### Example

```
git push -u origin master
```

This will upload the project to GitHub.

---

# 11. Git Clone Command

The `git clone` command creates a copy of an existing repository.

### Syntax

```
git clone URL
```

### Example

```
git clone https://github.com/username/devops-lab.git
```

This creates a local copy of the repository.

---

# 12. Git Reset Command

The `git reset` command is used to undo changes in the repository.

### Syntax

```
git reset --hard
```

### Example

```
git reset --hard HEAD
```

This resets the repository to the previous commit and removes changes.
