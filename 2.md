

# Requirements

* Git installed on the system
* GitHub account
* GitBash terminal
* Internet connection

---

# 1. Git Fork

A **Fork** is a copy of a repository. Forking allows you to experiment with changes without affecting the original repository.

## Steps to Fork a Repository

1. Login to **GitHub**.
2. Open the repository you want to fork.
3. Click the **Fork** button at the top-right corner.
4. GitHub creates a copy of the repository in your account.

Example:

```text
Original Repository → github.com/original-user/project
Forked Repository → github.com/your-username/project
```

---

# 2. Git Branch

A **branch** is a separate version of the repository that allows independent development.

## Create a Branch

### Syntax

```bash
git branch <branch-name>
```

### Example

```bash
git branch feature1
```

---

## Switch to Branch

```bash
git checkout feature1
```

---

## List All Branches

```bash
git branch
```

Output Example:

```text
* main
  feature1
```

---

## Delete a Branch

```bash
git branch -d feature1
```

---

# 3. Git Merge

Merging combines the work from different branches into a single branch.

### Syntax

```bash
git merge master
```

### Example

```bash
git checkout main
git merge feature1
```

This merges the **feature1 branch into the main branch**.

---

# 4. Git Diff

The `git diff` command compares changes between different file versions.

### Syntax

```bash
git diff
```

### Example Output

```text
- This is old content
+ This is updated content
```

This helps developers identify the differences between versions.

# 5. Git Merge Conflict

A **merge conflict** occurs when two branches modify the same part of the same file.

### Steps to Demonstrate Merge Conflict

Create a new directory:

```bash
mkdir git-merge-test
cd git-merge-test
```

Initialize repository:

```bash
git init
```

Create a file:

```bash
echo "this is some content to mess with" > merge.txt
```

Add and commit file:

```bash
git add merge.txt
git commit -am "we are committing the initial content"
```

Create a new branch:

```bash
git checkout -b new_branch_to_merge_later
```

Modify file:

```bash
echo "totally different content to merge later" > merge.txt
```

Commit changes:

```bash
git commit -am "edited the content of merge.txt to cause a conflict"
```

Merge branches:

```bash
git merge new_branch_to_merge_later
```

Example Output:

```text
Auto-merging merge.txt
CONFLICT (content): Merge conflict in merge.txt
Automatic merge failed; fix conflicts and then commit result
```

This indicates a **merge conflict**.

---

# 6. Resolve Merge Conflict

Open the file **merge.txt**.

You will see:

```text
<<<<<<< HEAD
this is some content to mess with
=======
totally different content to merge later
>>>>>>> new_branch_to_merge_later
```

Edit the file manually and keep the correct content.

Example:

```text
this is the final merged content
```

Then commit the changes:

```bash
git add merge.txt
git commit -m "Resolved merge conflict"
```

# 7. Git Pull

The **git pull** command downloads updates from a remote repository and merges them into the local repository.

### Syntax

```bash
git pull origin main
```

This command:

* Fetches changes from GitHub
* Merges them into the current branch

---

# 8. Pull Request

A **Pull Request (PR)** is used to request merging your changes into another branch or repository.

### Steps to Create Pull Request

1. Push your branch to GitHub.

```bash
git push origin feature1
```

2. Go to your repository on **GitHub**.
3. Click **Compare & Pull Request**.
4. Add description of the changes.
5. Click **Create Pull Request**.

The repository owner can review and merge the changes.
