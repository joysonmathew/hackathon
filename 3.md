

# 1. Installing Java Development Kit (JDK)

Jenkins requires Java to run. Therefore, JDK must be installed first.

## Steps

1. Download **JDK 8** from the official Oracle website.
2. Choose the correct version based on system configuration:

   * Windows 32-bit
   * Windows 64-bit
3. Accept the **License Agreement** and download the installer.
4. Install the JDK on your system.

## Setting Environment Variables

1. Open **System Properties**.
2. Go to **Advanced System Settings**.
3. Click **Environment Variables**.
4. Under **System Variables**, click **New**.

Add the following variables:

**JAVA_HOME**

```
Variable Name: JAVA_HOME
Variable Value: C:\Program Files\Java\jdk1.8.0_xxx
```

5. Edit the **PATH** variable.
6. Add the following path:

```
%JAVA_HOME%\bin
```

## Verify Java Installation

Open **Command Prompt** and run:

```
java -version
```

If installed correctly, Java version details will be displayed.

# 2. Download and Install Jenkins

## Steps

1. Go to the official Jenkins website.
2. Download **Jenkins LTS (Long Term Support)** for Windows.
3. Run the downloaded installer.
4. Follow the installation wizard.
5. Click **Finish** after installation.

---

# 3. Running Jenkins on Localhost

1. After installation, Jenkins automatically starts.
2. Open a web browser.
3. Enter the following URL:

```
http://localhost:8080
```

4. Jenkins will ask for an **initial admin password**.

Find the password from:

```
C:\Program Files\Jenkins\secrets\initialAdminPassword
```

5. Paste the password into Jenkins.

---

# 4. Install Recommended Plugins

After logging in:

1. Select **Install Suggested Plugins**.
2. Jenkins will automatically install required plugins.
3. Wait until installation completes.

# 6. Creating a Jenkins Job

## Steps

1. Click **New Item**.
2. Enter the project name:

```
Jenkins_demo
```

3. Select **Freestyle Project**.
4. Click **OK**.

### General Configuration

Add description:

```
This is my first Jenkins job
```

---

# 7. Build Step Configuration

1. Go to **Build Section**.
2. Click **Add Build Step**.
3. Select:

```
Execute Windows Batch Command
```

4. Enter the following command:

```
echo Hello... This is my first Jenkins Demo
echo Date: %date%
echo Time: %time%
```

5. Click **Apply**.
6. Click **Save**.

---

# 8. Running the Jenkins Job

1. Open the project.
2. Click **Build Now**.
3. A new build will start.

To view results:

1. Click the build number.
2. Click **Console Output**.

The output will display:

```
Hello... This is my first Jenkins Demo
Date: <current date>
Time: <current time>
```

# 9. Project Configuration with GitHub

## Step 1: Install Python

Download and install Python from:

```
https://www.python.org
```

Verify installation:

```
python --version
```

---

## Step 2: Create Python Script

Create a file named:

```
hello.py
```

Example program:

```python
print("Hello from Jenkins CI/CD Pipeline")
```

---

## Step 3: Push Project to GitHub

Initialize Git repository:

```
git init
```

Add files:

```
git add .
```

Commit changes:

```
git commit -m "Initial Commit"
```

Connect to GitHub repository:

```
git remote add origin https://github.com/username/repository-name.git
```

Push to GitHub:

```
git push -u origin main
```

# 10. Jenkins Configuration for GitHub Integration

## Step 1: Create New Project

1. Go to Jenkins Dashboard.
2. Click **New Item**.
3. Enter project name.
4. Select **Freestyle Project**.
5. Click **OK**.

---

## Step 2: Source Code Management

1. Go to **Source Code Management**.
2. Select **Git**.
3. Enter GitHub repository URL.

Example:

```
https://github.com/username/repository-name.git
```

4. Provide the branch name:

```
main
```

---

## Step 3: Build Trigger

Enable automatic build.

1. Select **Poll SCM**.
2. Configure schedule:

```
H/5 * * * *
```

This checks the repository every 5 minutes.

---

## Step 4: Build Step

Add build step.

1. Select **Execute Windows Batch Command**.
2. Enter:

```
python hello.py
```

---

## Step 5: Save and Build

1. Click **Apply**.
2. Click **Save**.
3. Click **Build Now**.

---

# 11. Verify Build Output

1. Open the build number.
2. Click **Console Output**.

Expected output:

```
Hello from Jenkins CI/CD Pipeline
```

If the script executes successfully, the **build status will be SUCCESS**.
